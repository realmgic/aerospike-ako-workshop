# Workshop secrets

## features.conf (never commit)

Copy your Aerospike Enterprise **feature-key file** here as `features.conf`:

```bash
cp /path/to/your/features.conf secrets/features.conf
```

This file is gitignored. Each instructor must supply their own license from the Aerospike licensing portal.

## Lab auth passwords (committed defaults)

The following are **generic throwaway credentials** for disposable EKS lab clusters. They are applied by [`scripts/setup/07-deploy-secrets.sh`](../scripts/setup/07-deploy-secrets.sh) and documented in lab guides:

| Kubernetes secret | User | Password |
|-------------------|------|----------|
| `auth-secret` | admin | `admin123` |
| `auth-app-secret` | app | `app123` |
| `auth-exporter-secret` | exporter | `exporter123` |

These are safe to commit and share within the private workshop repo.

## TLS/PKI material (Section 3 — never commit)

Generated by [`scripts/setup/tls/generate-workshop-pki.sh`](../scripts/setup/tls/generate-workshop-pki.sh) into `secrets/tls/` (gitignored):

| Local file | Kubernetes secret | Purpose |
|------------|-------------------|---------|
| `cacert.pem` / `cakey.pem` | `tls-ca-secret` | Workshop CA |
| `svc_chain.pem`, `svc_key.pem` | `tls-server-secret` | Service TLS (port 4333) |
| `admin.pem`, `admin.key` | `tls-client-admin-secret` | PKI user `admin` |
| `app.pem`, `app.key` | `tls-client-app-secret` | PKI user `app` / workload |
| `exporter.pem`, `exporter.key` | `tls-client-exporter-secret` | Exporter mTLS/PKI |
| `ako_client.pem`, `ako_client.key` | `tls-ako-client-secret` | AKO → DB (`operatorClientCert`) |

Deploy to the cluster with [`scripts/setup/tls/deploy-tls-secrets.sh`](../scripts/setup/tls/deploy-tls-secrets.sh).

Client certificate **CN must match** the Aerospike username for PKI authentication.
